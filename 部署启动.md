# lead_agent config 
入口是 LangGraph 图注册
在 langgraph.json:9 里把 lead_agent 指到 agent.py:265 的 make_lead_agent。
LangGraph 运行这个图时会把本次 run 的 config 传给它。

这个项目里 config 的一个明确构造点

是否“预定义”
分两层：

RunnableConfig 这个类型是框架预定义的（LangChain/LangGraph 类型）。
你在 configurable 里放哪些键（比如 is_plan_mode）是项目自己约定的，并不是框架强制固定字段。make_lead_agent 只是读取这些键（见 agent.py:270 和 agent.py:275）。
还有默认值来源
渠道管理器里有默认 run config/context，见 manager.py:19 到 manager.py:26。
所以最终是“默认值 + 调用方传入”合并后，传给 make_lead_agent。

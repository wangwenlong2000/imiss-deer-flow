# docker 部署启动
### 0. 前提条件
将当前用户加入docker 用户组
``` sh
sudo usermod -aG docker $USER
newgrp docker
```

### 1. 首先运行 下面的命令 后续这些文件的修改不会被git跟踪 也不会push到远程仓库
```sh
cd ./imiss-deerflow
git update-index --skip-worktree \
  backend/packages/harness/deerflow/community/aio_sandbox/aio_sandbox_provider.py \
  docker/nginx/nginx.conf \
  docker/nginx/nginx.local.conf \
  docker/docker-compose-dev.yaml \
  docker/docker-compose.yaml \
  scripts/cleanup-containers.sh \
  scripts/deploy.sh \
  scripts/docker.sh \
  scripts/serve.sh \
  scripts/start-daemon.sh \
  config.yaml
```

### 2. 修改 config.yaml文件里的端口号配置，改成自己的端口以及subnet ：
下面是原本的默认值 需要改成自己的端口号和subnet，避免和其他用户冲突：
4个端口和subnet都要改
```yaml
# Docker Port Overrides
# ============================================================================
# Used by scripts/update-docker-ports.sh to update docker/nginx related files.
# Priority when running the script: environment variables > docker_ports > built-in defaults.
docker_ports:
  nginx: 2026
  langgraph: 2024
  gateway: 8001
  frontend: 3000
subnet:
  dev: 192.168.200.0/24  # Docker bridge subnet for dev compose; must be unique per user on shared servers
```

### 3. make docker-update-ports-cname 更新docker 构建启动脚本
执行 make docker-update-ports-cname
这一步会修改 容器的端口配置 镜像名 容器名 
具体实现在 scripts/update-docker-ports-cname.sh


### 4. make docker-init 拉取sandbox 镜像
```bash
make docker-init    # Pull sandbox image (only once or when image updates)
``` 

### 以上命令只需要执行一次  配置完成后 后续只需要执行 make docker-start 和 make docker-stop 来启动和关闭容器即可
-----------------------------------------------------

### 5. make docker-start 启动容器 等待启动 即可通过nginx端口 使用浏览器访问deer-flow
这里启动的容器有热重载服务 容器内运行的命令会监听代码变化 自动重启服务 方便开发调试
```bash
make docker-start   # Start containers (after init)
```


### 6. make docker-stop 关闭容器 修改特定的配置文件后需要重启容器才能生效 这时可以先执行 make docker-stop 关闭容器 然后再执行 make docker-start 启动容器
```bash
make docker-stop   # Stop containers
```

## 端口说明

用户是直接访问 Frontend
访问地址就是脚本输出的前端端口，例如 3000 或回退到 3001，见 dev-no-nginx.sh:192。

Front 和 Backend 是直接通信
脚本在启动前端时，直接注入了后端地址：
dev-no-nginx.sh:178
dev-no-nginx.sh:179

所以浏览器会直接请求：

补充一点：
前端代码本身支持两种模式。若不设置 LangGraph 环境变量，会走同源的 /api/langgraph（这是给 Nginx 反代模式准备的），见 index.ts:22。你现在脚本已显式设置了直连地址，因此不会依赖 Nginx 路由。




# lead_agent config 
入口是 LangGraph 图注册
在 langgraph.json:9 里把 lead_agent 指到 agent.py:265 的 make_lead_agent。
LangGraph 运行这个图时会把本次 run 的 config 传给它。

这个项目里 config 的一个明确构造点
在 client.py:170 的 _get_runnable_config 会组装 RunnableConfig，包含 configurable 里的 thread_id、model_name、thinking_enabled、is_plan_mode、subagent_enabled，随后在 client.py:306 调用时传入。

是否“预定义”
分两层：

RunnableConfig 这个类型是框架预定义的（LangChain/LangGraph 类型）。
你在 configurable 里放哪些键（比如 is_plan_mode）是项目自己约定的，并不是框架强制固定字段。make_lead_agent 只是读取这些键（见 agent.py:270 和 agent.py:275）。
还有默认值来源
渠道管理器里有默认 run config/context，见 manager.py:19 到 manager.py:26。
所以最终是“默认值 + 调用方传入”合并后，传给 make_lead_agent。

## 本地 JSONL 日志怎么查看
日志目录由 DEERFLOW_RUN_EVENT_LOG_DIR 控制。

如果使用当前默认配置，目录是：

```bash
/nfsdat/home/akzhaoslm/deer-flow/logs/run_events
```

常用命令：

查看目录：

```bash
ls /nfsdat/home/akzhaoslm/deer-flow/logs/run_events
```

查看某个线程最近 50 行：

```bash
tail -n 50 /nfsdat/home/akzhaoslm/deer-flow/logs/run_events/<thread_id>.jsonl
```

筛选 agent 最终结果：

```bash
grep '"event": "agent.run.final"' /nfsdat/home/akzhaoslm/deer-flow/logs/run_events/<thread_id>.jsonl
```

筛选非流式结果：

```bash
grep '"event": "runs.wait.result"' /nfsdat/home/akzhaoslm/deer-flow/logs/run_events/<thread_id>.jsonl
```

筛选流式结束结果：

```bash
grep '"event": "runs.stream.final"' /nfsdat/home/akzhaoslm/deer-flow/logs/run_events/<thread_id>.jsonl
```

## LangSmith 怎么查看
1. 在 .env 中配置 LANGSMITH_TRACING、LANGSMITH_API_KEY、LANGSMITH_PROJECT。
2. 重启服务。
3. 发起一次真实对话、工具调用或上传操作。
4. 打开 LangSmith 控制台，进入 deer-flow 项目查看 traces。
5. 点击链接打开https://smith.langchain.com/

## 本地 JSONL 当前记录的事件
当前会记录以下事件：

- agent.run.final
- runs.wait.result
- runs.stream.start
- runs.stream.chunk
- runs.stream.error
- runs.stream.final

它们都会按 thread_id 分文件写入，适合排查：

- 文件上传相关问题
- Feishu 流式输出问题
- 非流式 wait 返回异常
- agent 最终状态与运行上下文


## 监控能力说明
当前项目支持两套监控方案：

1. LangSmith
用于查看模型调用链、trace、运行过程。

2. 本地 JSONL
用于按 thread_id 把关键运行事件落盘到本机，方便直接 tail、grep、排查问题。

两套可以同时开启，也可以分别单独开启。

## 当前使用的环境变量
LangSmith：

- LANGSMITH_TRACING
- LANGSMITH_API_KEY
- LANGSMITH_PROJECT
- LANGSMITH_ENDPOINT

本地 JSONL：

- DEERFLOW_RUN_EVENT_LOG_ENABLED
- DEERFLOW_RUN_EVENT_LOG_DIR

## 推荐配置示例
两套都开启：

```env
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=your-langsmith-api-key
LANGSMITH_PROJECT=deer-flow
LANGSMITH_ENDPOINT=https://api.smith.langchain.com

DEERFLOW_RUN_EVENT_LOG_ENABLED=1
DEERFLOW_RUN_EVENT_LOG_DIR=/nfsdat/home/akzhaoslm/deer-flow/logs/run_events
```

只开 LangSmith：

```env
LANGSMITH_TRACING=true
DEERFLOW_RUN_EVENT_LOG_ENABLED=0
```

只开本地 JSONL：

```env
LANGSMITH_TRACING=false
DEERFLOW_RUN_EVENT_LOG_ENABLED=1
```

全部关闭：

```env
LANGSMITH_TRACING=false
DEERFLOW_RUN_EVENT_LOG_ENABLED=0
```

关闭 LangSmith：


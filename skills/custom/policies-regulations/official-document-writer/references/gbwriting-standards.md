# 国家机关公文写作规范（GB/T 9704-2012）

> 本文件依据《党政机关公文格式》国家标准 GB/T 9704-2012 整理，为公文排版的权威依据。

## 一、公文格式精确排版参数

### 1. 纸张与版心
| 参数 | 数值 |
|------|------|
| 用纸规格 | A4（210mm × 297mm） |
| 版心尺寸 | 156mm × 225mm（不含页码） |
| 上白边（页边距） | 37mm ± 1mm |
| 下白边 | 35mm ± 1mm |
| 左白边 | 28mm ± 1mm |
| 右白边 | 26mm ± 1mm |
| 每面行数 | 22行 |
| 每行字数 | 28字 |
| 行距 | 固定值 28磅（560 twips） |

### 2. 字体字号精确规范
| 公文要素 | 字体 | 字号 | 对齐 | 缩进 |
|---------|------|------|------|------|
| 公文标题 | 小标宋体（方正小标宋/宋体粗） | **2号（22pt）** | 居中 | 无 |
| 一级标题（一、二、三……） | **黑体** | 3号（16pt） | 两端对齐 | 顶格（无缩进） |
| 二级标题（（一）（二）……） | **楷体_GB2312** | 3号（16pt） | 两端对齐 | 首行缩进2字 |
| 三级标题（1. 2. 3.） | 仿宋_GB2312 | 3号（16pt） | 两端对齐 | 首行缩进2字 |
| 正文 | **仿宋_GB2312** | 3号（16pt） | 两端对齐 | 首行缩进2字 |
| 主送机关 | 仿宋_GB2312 | 3号（16pt） | 两端对齐 | 顶格（无缩进） |
| 发文字号 | 仿宋_GB2312 | 3号（16pt） | 居中 | 无 |
| 签发人姓名 | 楷体_GB2312 | 3号（16pt） | 右对齐 | 无 |
| 成文日期 | 仿宋_GB2312 | 3号（16pt） | 右对齐 | **右空四字（64pt）** |
| 发文机关署名 | 仿宋_GB2312 | 3号（16pt） | 右对齐 | **右空四字（64pt）** |
| 页码 | 宋体半角 | **4号（14pt）** | 单页居右 | 距版心下边缘7mm |

### 3. 关键间距规定
- **标题与主送机关**：标题下空一行（即一个28pt行高）
- **一级标题前**：上方空一行行距
- **结束语（特此报告）前**：空一行
- **署名前**：空两行
- **署名与日期**：紧邻，不空行
- **成文日期格式**：年月日用阿拉伯数字，不补零（如 2026年3月30日）

### 2. 公文各要素排列规则

#### 标题
- 由发文机关名称、事由和文种三部分组成
- 一般用方正小标宋简体二号字，居中排列
- 可分一行或多行居中排布；回行时词意完整、排列对称、间距恰当
- 长标题：合理拆分行，避免出现"的"字孤行
- 标题中的标点：一般不加标点符号

#### 主送机关
- 标题下空一行，左顶格书写
- 多个机关之间用顿号或逗号分隔
- 回行顶格，最后一个名称后加全角冒号

#### 正文
- 首行缩进2个中文字符
- 仿宋_GB2312三号字
- 层次序号依次使用：
  - 第一层：一、二、三、（黑体字）
  - 第二层：（一）（二）（三）（楷体字）
  - 第三层：1. 2. 3.（仿宋字）
  - 第四层：（1）（2）（3）（仿宋字）

#### 落款与日期
- 成文日期右空四字
- 用阿拉伯数字将年、月、日标全
- 发文机关署名在成文日期上方

### 3. 页码
- 用四号半角阿拉伯数字
- 左右各放一条一字线
- 单页码居右空一字，双页码居左空一字

## 二、常用公文文种规范

### 请示（上行文）

#### 基本规范
- **定义**：向上级机关请求指示、批准
- **适用场景**：
  - 请求上级对某项工作给予指示
  - 请求批准某个项目、方案
  - 请求拨付资金、调配物资
  - 请求批准机构设置、人事变动
- **核心规则**：
  - **一文一事**：一个请示只请示一件事
  - **单头请示**：主送机关只有一个
  - **不越级**：原则上逐级行文
  - **事前行文**：在问题处理前请示

#### 结构框架
```
标题：[发文机关]关于[事由]的请示
主送机关：[上级机关名称]

[请示缘由]
说明请示事项的背景、原因、必要性
- 政策依据
- 现状分析
- 问题的紧迫性

[请示事项]
具体说明需要上级批准或指示的内容
- 方案要点
- 实施计划
- 所需资源

[请示语]
妥否，请批示。

[发文机关署名]
[成文日期]
```

### 报告（上行文）

#### 基本规范
- **定义**：向上级机关汇报工作、反映情况、回复上级机关询问
- **适用场景**：
  - 定期汇报工作进展
  - 专题汇报某项工作开展情况
  - 反映重大情况或问题
  - 回复上级机关的询问
- **核心规则**：
  - **不得夹带请示事项**：报告中不能要求上级批准什么
  - **实事求是**：情况真实、数据准确
  - **突出重点**：抓住关键、言简意赅
  - **层次分明**：条理清晰、逻辑严密

#### 结构框架
```
标题：[发文机关]关于[事由]的报告
主送机关：[上级机关名称]

[开头段]
简要说明报告的目的和背景
（"根据……精神，现将……情况报告如下："）

[正文主体]
一、基本情况
  （一）主要成效
  （二）主要做法
二、存在问题
三、下一步工作打算

特此报告。

[发文机关署名]
[成文日期]
```

## 三、标题撰写规范

### 1. 标题对仗工整的要求
- **结构对齐**：各分句结构一致（名词+动词+名词）
- **字数相当**：各分句字数相同或相近
- **意义关联**：各分句意义互补、逻辑递进
- **语体统一**：使用公文语体，保持庄重

### 2. 标题常见模式
#### 并列式
- "关于深化改革创新 推动高质量发展的意见"
- "关于加强制度建设 规范管理流程的通知"

#### 递进式
- "关于深化XX改革 加快推进XX建设的请示"
- "关于全面提升XX水平 努力实现XX目标的报告"

#### 因果式
- "关于适应新形势要求 进一步做好XX工作的意见"

### 3. 标题用词规范
- **避免**：口语化词汇、网络用语、过于文学化的表达
- **推荐**："深入推进""全面加强""着力提升""切实抓好""进一步"
- **数量词规范**：使用汉字数字（"一个""两项""三个"）

## 四、正文写作技术规范

### 1. 段落结构
- 每段一般不超过200字
- 段首不使用空格，用首行缩进2字符
- 段间逻辑紧密，注意过渡衔接

### 2. 数字使用规范
- **公历世纪、年代、年月日**：用阿拉伯数字（2026年3月）
- **概数**：用汉字（三四个月、七八十人）
- **法规文件编号**：用阿拉伯数字（国发〔2026〕X号）
- **百分比**：用阿拉伯数字（增长率12.5%）

### 3. 引用规范
- 引用公文先引标题，后引发文字号
- 格式："《XXX文件》（X发〔2026〕X号）"
- 引用领导讲话："按照X部长在X会议上讲话精神"
- 引用政策原文需加引号并标注出处

### 4. 附件说明
- 附件名称前加"附件："二字
- 如有多个附件，用阿拉伯数字编号
- 附件名称后不加标点符号

## 五、常见格式错误与规避

| 错误类型 | 正确做法 |
|---------|---------|
| 标题用词口语化 | 使用公文标准用语 |
| 层次序号混乱 | 严格按一、（一）、1、（1）递进 |
| 数字使用不规范 | 遵守GB/T 15835数字用法标准 |
| 引用缺少出处 | 标注完整文件名与文号 |
| 请示一文多事 | 一个请示只涉及一个事项 |
| 报告夹带请示 | 报告中不得要求上级批准事项 |
| 签发日期用汉字 | 用阿拉伯数字标注日期 |
| 标题回行断词不当 | 回行时保持词意完整 |

## 六、篇幅控制要求

### 5页A4纸标准（约3500-4500字）
- 标题 + 主送机关：约50字
- 正文主体：约3000-4000字
  - 开头段（背景/目的）：200-400字
  - 核心内容（3-4个层次）：2000-3000字
  - 结尾段（下一步打算/请示语）：200-300字
- 落款与日期：约30字

### 各部分篇幅建议
- 简短报告（2-3页）：1500-2500字
- 标准报告/请示（4-5页）：3000-4500字
- 综合报告（不超过5页上限）：4000-5000字

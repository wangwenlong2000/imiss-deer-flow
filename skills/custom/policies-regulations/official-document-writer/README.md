# 公文撰写助手 (Official Document Writer)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![OpenClaw Skill](https://img.shields.io/badge/OpenClaw-Skill-blue)](https://openclaw.ai/)

> 自动搜索最新国家政策文件和主管部门领导讲话精神，严格按照 GB/T 9704-2012 国家公文格式标准，生成逻辑严谨、语言精炼、标题对仗工整的请示或报告，输出 Word 和 PDF 双格式。

## ✨ 功能特性

- 🏛️ **政策先行**：自动搜索国务院、部委最新政策文件和领导讲话
- 📐 **国标排版**：严格遵循 GB/T 9704-2012 公文格式标准
- 📝 **双格式输出**：同时生成 Word (.docx) 和 PDF 文件
- 🎨 **标题精工**：自动生成对仗工整、结构对称的公文标题
- 📊 **篇幅控制**：智能控制在 4-5 页 A4 纸（3500-4500 字）

## 📋 支持的公文文种

| 文种 | 说明 | 典型用途 |
|------|------|---------|
| **请示** | 上行文，请求上级批准/指示 | 申请资金、请求批复、人事任免 |
| **报告** | 上行文，汇报工作/反映情况 | 工作总结、情况汇报、调研报告 |

## 📐 GB/T 9704-2012 排版规范

本技能严格遵循《党政机关公文格式》国家标准：

| 要素 | 字体 | 字号 | 对齐 | 缩进 |
|------|------|------|------|------|
| 公文标题 | 小标宋体 | 2号（22pt） | 居中 | 无 |
| 一级标题（一、二、三） | 黑体 | 3号（16pt） | 两端对齐 | 顶格 |
| 二级标题（（一）（二）） | 楷体_GB2312 | 3号（16pt） | 两端对齐 | 首行缩进2字 |
| 正文 | 仿宋_GB2312 | 3号（16pt） | 两端对齐 | 首行缩进2字 |
| 主送机关 | 仿宋_GB2312 | 3号（16pt） | 两端对齐 | 顶格 |
| 署名/日期 | 仿宋_GB2312 | 3号（16pt） | 右对齐 | 右空四字 |

**版心参数**：
- 纸张：A4（210mm × 297mm）
- 页边距：上37mm / 下35mm / 左28mm / 右26mm
- 版心：156mm × 225mm
- 每面22行，每行28字
- 行距：固定值28磅

## 🔍 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│ 第一阶段：需求确认与信息收集                                  │
│  ├── Step 1: 明确需求（文种、发文机关、主送机关、主题）        │
│  ├── Step 2: 搜索政策依据（国务院、部委最新文件）             │
│  └── Step 3: 整理政策素材                                    │
├─────────────────────────────────────────────────────────────┤
│ 第二阶段：起草公文                                           │
│  ├── Step 4: 确定框架（请示/报告结构）                       │
│  ├── Step 5: 拟定标题（2-3个方案供选择）                     │
│  └── Step 6: 撰写正文（严格遵循格式规范）                    │
├─────────────────────────────────────────────────────────────┤
│ 第三阶段：润色校对                                           │
│  ├── Step 7: 质量自检（10项检查清单）                        │
│  └── Step 8: 生成文档（Word + PDF 双格式）                   │
├─────────────────────────────────────────────────────────────┤
│ 第四阶段：交付与说明                                         │
│  └── Step 9: 交付成果（文件路径、政策清单、结构概览）        │
└─────────────────────────────────────────────────────────────┘
```

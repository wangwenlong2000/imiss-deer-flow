# 发明专利初稿模板

> 本模板用于在技术交底书、权利要求布局卡和权利要求-代码证据矩阵基础上，继续生成接近可申报版的中国发明专利初稿。
> 使用前应先完成候选方案确认、项目分析、代码证据映射和 L2.5 布局产物，避免把未实现内容误写为既有实现。
> 默认起草口径：方法 + 系统 / 装置 + 计算机可读存储介质 + 计算机程序产品四类保护主题草稿，正式提交前仍需由专利代理师结合审查策略整合。

## 二、权利要求书草稿

> 默认输出四类平行保护主题草稿，供代理师择优整合。正式编号和最终保留范围由代理师决定。

### 1. 方法独立权利要求

一种[方法名称]，其特征在于，包括：

1. [步骤或特征 1]；
2. [步骤或特征 2]；
3. [步骤或特征 3]；
4. [步骤或特征 4]。

### 2. 系统 / 装置独立权利要求

一种[系统 / 装置名称]，其特征在于，包括：

- [模块 / 单元 1]，用于[对应方法步骤 1]；
- [模块 / 单元 2]，用于[对应方法步骤 2]；
- [模块 / 单元 3]，用于[对应方法步骤 3]。

### 3. 计算机可读存储介质独立权利要求

一种计算机可读存储介质，其上存储有计算机程序，其特征在于，所述计算机程序被处理器执行时实现如权利要求 1 所述的方法。

### 4. 计算机程序产品独立权利要求

一种计算机程序产品，其特征在于，包括计算机程序，所述计算机程序被处理器执行时实现如权利要求 1 所述的方法。

### 5. 从属权利要求

#### 5.1 挂接方法独立权利要求

2. 根据权利要求 1 所述的方法，其特征在于，[进一步限定核心增强特征]。

3. 根据权利要求 1 所述的方法，其特征在于，[进一步限定优选参数 / 条件]。

4. 根据权利要求 1 所述的方法，其特征在于，[进一步限定缓存 / 回退 / 鉴权 / 调度细节]。

#### 5.2 挂接系统 / 装置独立权利要求

5. 根据前述系统 / 装置独立权利要求所述的系统 / 装置，其特征在于，[进一步限定模块关系或协同逻辑]。

#### 5.3 挂接存储介质独立权利要求

6. 根据前述计算机可读存储介质独立权利要求所述的计算机可读存储介质，其特征在于，[进一步限定程序执行条件或实现细节]。

#### 5.4 挂接计算机程序产品独立权利要求

7. 根据前述计算机程序产品独立权利要求所述的计算机程序产品，其特征在于，[进一步限定程序模块、部署方式或执行约束]。

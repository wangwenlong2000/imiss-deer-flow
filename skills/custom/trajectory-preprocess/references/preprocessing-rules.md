# 轨迹预处理规则

## 清洗规则

- 丢弃缺失用户、时间、经纬度的记录。
- 丢弃纬度不在 `[-90, 90]` 或经度不在 `[-180, 180]` 的记录。
- 同一用户、同一轨迹、同一时间、同一坐标的重复点只保留一条。
- 按 `user_id + trajectory_id + timestamp` 排序后处理。

## 去噪规则

默认使用速度阈值过滤异常跳点。

- 距离使用 Haversine 公式。
- 速度 = 相邻点距离 / 时间间隔。
- 若速度超过 `--max-speed-kmh`，当前点标记为异常并剔除。
- 默认阈值为 `200 km/h`，适合城市尺度 GPS/check-in 数据。

## 停留点规则

默认停留点算法：

- 从某用户轨迹中的一个点开始向后扫描。
- 如果后续点一直位于起点 `--stay-radius-m` 半径内，并且停留时长超过 `--stay-min-minutes`，则生成停留点。
- 停留点位置为窗口内点的质心。

默认参数：

- `--stay-radius-m 200`
- `--stay-min-minutes 20`

## 轨迹切分规则

默认按时间间隔切分出行段：

- 同一用户或同一原始轨迹内，如果相邻点时间间隔超过 `--trip-gap-minutes`，则开启新 trip。
- 少于 `--min-trip-points` 的片段不输出。

默认参数：

- `--trip-gap-minutes 60`
- `--min-trip-points 2`

## 空间编码

- 默认输出 `geohash`。
- 默认精度为 `6`。
- 可以通过 `--geohash-precision` 调整。

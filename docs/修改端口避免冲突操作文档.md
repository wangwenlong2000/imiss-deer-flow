# 修改 DeerFlow 端口避免冲突操作文档

## 背景

当多台用户在同一台服务器上运行 DeerFlow 时，默认都会使用 **2026** 端口，会产生端口冲突。本文档介绍如何修改端口避免冲突。

## 端口说明

DeerFlow 的端口架构分为两部分：

| 层级 | 端口 | 说明 |
|------|------|------|
| **主机端口** | 2026（默认）| 外部浏览器访问的端口，可自定义修改 |
| **容器内部端口** | 2026 | Nginx 容器内部监听的端口，**不建议修改** |

**推荐做法**：只修改主机端口映射，保持容器内部端口不变。

---

## 方法一：使用环境变量快速修改（推荐）

最简单的方法，无需修改配置文件：

```bash
# 启动时指定自定义 PORT 环境变量
PORT=2028 make docker-start

# 或者生产模式
PORT=2028 make up
```

访问地址变为：`http://localhost:2028`

> **注意**：此方法每次启动都需要指定 PORT，重启后失效。

---

## 方法二：修改配置文件永久生效

### 步骤 1：修改 docker-compose-dev.yaml

编辑文件：`docker/docker-compose-dev.yaml`

找到 nginx 服务的 ports 配置（约第 67 行）：

```yaml
  nginx:
    image: nginx:alpine
    container_name: deer-flow-nginx
    ports:
      - "2028:2026"    # 将左侧的 2028 改为你想要的端口
```

**格式说明**：`"主机端口:容器内部端口"`

- **左侧（2027）**：改为你想要的主机端口
- **右侧（2026）**：保持 2026 不变

### 步骤 2：修改生产环境配置（如果使用 make up）

编辑文件：`docker/docker-compose.yaml`

找到 nginx 服务的 ports 配置（约第 30 行）：

```yaml
  nginx:
    ports:
      - "${PORT:-2028}:2026"   # 将 2028 改为你想要的端口
```

同样只修改 `:-` 后面的默认值，保持容器端口 2026 不变。

### 步骤 3：重启服务

```bash
# 停止旧服务
make docker-stop

# 启动新服务
make docker-start
```

## 验证端口是否生效

### 1. 查看容器端口映射

```bash
docker ps --filter "name=deer-flow-nginx" --format "table {{.Names}}\t{{.Ports}}"
```

输出示例：
```
NAMES               PORTS
deer-flow-nginx     80/tcp, 0.0.0.0:2028->2026/tcp, [::]:2028->2026/tcp
```

看到 `2028->2026` 表示配置成功。

### 2. 浏览器访问

打开浏览器访问：`http://localhost:<你的端口>`

### 3. 检查端口是否被占用

```bash
# 查看某个端口是否被占用
lsof -i :2028

# 或者
netstat -tlnp | grep 2028
```

## 常见问题

### Q1：修改端口后前端无法访问？

**原因**：只修改了主机端口，但容器内部 nginx 仍监听 2026。

**解决**：确保端口映射格式为 `"新端口:2026"`，不要修改右侧的 2026。

### Q2：可以使用任意端口吗？

建议使用 **1024~65535** 之间的端口，避免与系统端口冲突。

常用备选端口：
- 2028, 2029, 2030
- 3000, 3001, 3002
- 8080, 8081, 8082

### Q3：多人同时使用会不会互相影响？

不会。每个人使用不同的主机端口，完全隔离：
- 用户 A 访问 http://localhost:2026
- 用户 B 访问 http://localhost:2028
- 各自的数据和容器互不干扰

### Q4：如何完全停止服务？

```bash
# 开发模式
make docker-stop

# 生产模式
make down
```

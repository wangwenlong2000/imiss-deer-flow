# 网络流量分析示例问题清单

这份文档整理了当前系统已经比较支持的网络流量分析提问方式，便于本地测试、能力验收和后续对外说明。

## 1. 总览类

这类最稳，适合直接得到整体统计结果。

示例：

- `使用 Gmail.flow.csv，帮我做一次网络流量总览分析，给出记录数、总字节数、总包数、源 IP 数、目的 IP 数、协议分布。`
- `使用 Outlook.pcap，先预处理，再给我这份流量的整体概况。`

## 2. TopN 排名类

适合查最活跃的源 IP、目的 IP、端口或协议。

示例：

- `使用 Gmail.flow.csv，分析按字节数排名前 10 的源 IP。`
- `使用 Outlook.flow.csv，统计最常见的前 10 个目的端口以及对应流量。`
- `使用 Geodo.pcap，先预处理，再找出访问目的 IP 数量最多的前 20 个源 IP。`

## 3. 分布类

适合看端口、协议、应用协议、服务分布。

示例：

- `使用 Gmail.flow.csv，统计目的端口分布。`
- `使用 Outlook.flow.csv，统计协议分布，并给出每类协议的记录数和总字节数。`
- `使用处理后的 pcap 结果，统计 app_protocol 分布。`

## 4. 时间趋势类

适合看流量高峰和时间变化。

示例：

- `使用 Gmail.flow.csv，按小时统计总字节数变化。`
- `使用 Outlook.flow.csv，按分钟聚合流量，看看是否存在明显峰值。`

## 5. 过滤与导出类

适合抽取特定流量子集，或导出结果文件。

示例：

- `使用 Gmail.flow.csv，筛出目的端口为 80 或 443 的流量。`
- `使用 Outlook.flow.csv，把 tls_sni 不为空的流量导出成 CSV。`
- `使用 Geodo.flow.csv，筛选 src_ip 为 1.2.3.4 的所有记录。`

## 6. 聚合类

适合按多个维度做汇总统计。

示例：

- `使用 Gmail.flow.csv，按 src_ip 和 protocol 分组统计记录数与总字节数。`
- `使用 Outlook.flow.csv，按 dst_port 聚合统计 packets 和 bytes。`

## 7. SQL 查询类

这类已经支持，而且现在因为字段扩展，查询可以更细。

示例：

- `使用 Gmail.flow.csv，执行 SQL：SELECT src_ip, COUNT(*) AS flows, SUM(bytes) AS total_bytes FROM flows GROUP BY src_ip ORDER BY total_bytes DESC LIMIT 10`
- `使用 Outlook.flow.csv，执行 SQL：SELECT dst_port, app_protocol, service, COUNT(*) AS flows FROM flows GROUP BY 1,2,3 ORDER BY flows DESC LIMIT 20`
- `使用处理后的 pcap 结果，执行 SQL：SELECT dns_query, COUNT(*) AS flows FROM flows WHERE dns_query IS NOT NULL GROUP BY dns_query ORDER BY flows DESC LIMIT 20`

## 8. 基础异常检测类

当前是规则型筛查，不是高级检测器，但适合做第一轮检查。

示例：

- `使用 Gmail.flow.csv，检查是否存在可疑扫描源 IP。`
- `使用 Outlook.flow.csv，找出稀有目的端口流量。`
- `使用某个 pcap 预处理后的流量，检查是否存在流量突增。`

## 9. PCAP 预处理 + 分析类

这是当前新增的重要能力，适合直接从抓包开始提问。

示例：

- `使用 Outlook.pcap，先预处理，统计目的端口分布。`
- `使用 Geodo.pcap，先预处理，再分析是否存在异常扫描行为。`
- `使用 corp-day1.pcap 和 corp-day2.pcap，先一起预处理，再比较协议分布差异。`

## 10. 基于扩展字段的查询

这部分现在比之前更强，适合查应用层相关字段。

示例：

- `使用某个 flow.csv，统计 tls_sni 分布。`
- `使用某个 flow.csv，列出 http_host 不为空的前 20 条流量。`
- `使用某个 flow.csv，按 traffic_family 统计流量。`
- `使用某个 flow.csv，查询 session_state 为 RST 的流量。`

## 一句话总结

当前已经比较支持的核心能力是：

**预处理 + 统计 + 查询 + 基础异常检测 + 导出**

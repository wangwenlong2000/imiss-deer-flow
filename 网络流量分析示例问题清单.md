# 网络流量分析示例问题清单

这份清单分成两部分：

- 第一部分：更接近真实用户会说的话
- 第二部分：更适合专业分析师、测试和验收的字段型问法

这样既能覆盖真实产品场景，也能保留专业分析场景。

使用原则：

- 对 `flow.csv`、`packet.csv` 这类结构化结果文件，更适合用“阅读”
- 对 `pcap`、`pcapng`、`cap` 这类原始抓包文件，更适合用“分析”，并明确“先预处理”
- 普通用户不需要知道字段名，系统应根据问题意图自动选择合适的分析动作
- 专业用户可以直接用字段或规则名称提问

## 第一部分：真实用户问法

### 1. 整体概况

- `阅读 Gmail.flow.csv，给我一个整体概况。`
- `分析 Htbot.pcap，先预处理，再告诉我这份流量的大致情况。`
- `阅读 Virut.flow.csv，给我一份企业级流量总览。`
- `分析 Zeus.pcap，先预处理，再给我总览报告。`

### 2. 主要通信对象

- `阅读 Outlook.flow.csv，哪些主机最活跃？`
- `阅读 Skype.flow.csv，主要在和哪些目标通信？`
- `阅读 Gmail.flow.csv，哪些端口最常见？`
- `阅读 BitTorrent.flow.csv，这份流量主要是什么类型的通信？`

### 3. 协议和应用类型

- `阅读 Outlook.flow.csv，这份流量主要用了哪些协议？`
- `阅读 Gmail.flow.csv，里面最常见的应用层协议是什么？`
- `阅读 Virut.flow.csv，看看有没有比较明显的 DNS、TLS 或 HTTP 特征。`
- `阅读 Geodo.packet.csv，从包级视角看看协议特征。`

### 4. 会话质量和连接结果

- `阅读 Outlook.flow.csv，这份流量里连接质量怎么样？`
- `阅读 Zeus.flow.csv，有没有很多失败连接？`
- `阅读 Virut.flow.csv，有没有很多异常短连接？`
- `阅读 Geodo.packet.csv，看一下是否存在握手异常或大量重置连接。`

### 5. 扫描和探测行为

- `阅读 Geodo.flow.csv，这份流量里有没有扫描行为？`
- `阅读 Geodo.packet.csv，从包级别检查有没有 SYN-only 扫描。`
- `阅读 Zeus.packet.csv，有没有主机在探测很多端口？`
- `阅读 Virut.flow.csv，先帮我筛一下可疑扫描源。`
- `阅读 Virut.flow.csv，如果发现可疑对象，再继续看 packet 级证据。`

### 6. packet 级专项分析

- `阅读 Virut.packet.csv，给我一个 packet 层的诊断摘要。`
- `阅读 Zeus.packet.csv，看看 TCP flags 有什么特点。`
- `阅读 Outlook.packet.csv，有没有 reset-heavy 的异常主机？`
- `阅读 Geodo.packet.csv，检查有没有握手失败模式。`
- `阅读某个 packet.csv，看看有没有 ICMP 探测。`
- `阅读某个 packet.csv，检查是否存在异常的小包突发。`

### 7. 异常检测

- `阅读 FTP.flow.csv，这份流量正常吗？有没有异常端口访问？`
- `阅读 Outlook.flow.csv，有没有明显的异常流量高峰？`
- `阅读 Zeus.flow.csv，这份流量里最值得关注的异常是什么？`
- `阅读 Virut.flow.csv，有没有可疑的主机行为？`
- `阅读 Zeus.packet.csv，有没有很多只发起不完成的连接？`
- `阅读 Virut.packet.csv，有没有很多异常重置连接？`
- `阅读某个 packet.csv，有没有探测式的 ICMP 行为？`
- `阅读某个 packet.csv，是否存在明显异常的小包洪峰？`

### 8. 时间趋势

- `阅读 Gmail.flow.csv，帮我看看流量随时间的变化。`
- `阅读 Outlook.flow.csv，有没有某个时间段特别异常？`
- `阅读 BitTorrent.flow.csv，按小时看流量趋势。`
- `阅读某个 flow.csv，找出可能的突发窗口。`

### 9. 资产和通信关系

- `阅读 Gmail.flow.csv，哪些主机连接的目标最多？`
- `阅读 Outlook.flow.csv，哪些目标被访问得最频繁？`
- `阅读某个 flow.csv，哪些设备最活跃？`
- `阅读某个 flow.csv，哪些用户或资产最值得关注？`
- `阅读某个 flow.csv，通信关系最复杂的主机有哪些？`

### 10. 深入调查

- `阅读 Gmail.flow.csv，帮我进一步调查最活跃的前几个源 IP。`
- `阅读 Outlook.flow.csv，围绕最主要的几个端口做深入分析。`
- `阅读某个 flow.csv，帮我找出最值得继续深挖的异常对象。`
- `阅读某个 packet.csv，围绕一个可疑主机继续做包级调查。`

### 11. 导出和筛选

- `阅读 Outlook.flow.csv，把和 Web 访问相关的记录导出来。`
- `阅读 Gmail.flow.csv，把有 DNS 查询的记录单独导出来。`
- `阅读 Geodo.flow.csv，只保留某个源 IP 相关的流量。`
- `阅读某个 packet.csv，把只发 SYN 的包单独导出。`

### 12. 上传文件与本地数据集

- `分析 Htbot.pcap，先预处理，再给我这份流量的整体概况。`
- `阅读 Geodo.flow.csv，看看最活跃的源 IP 是哪些。`
- `我刚上传了 Outlook.flow.csv，请优先分析上传文件，不要用本地同名数据集。`
- `阅读本地数据集 Virut.packet.csv，检查有没有 SYN-only 扫描。`

### 13. 企业级组合问法

- `阅读 Outlook.flow.csv，先给我一份整体总览，再补充扫描风险、连接质量和协议特征。`
- `分析 Zeus.pcap，先预处理，再依次做总览、异常筛查和会话质量分析。`
- `阅读 Virut.flow.csv，先做总体排查；如果发现可疑对象，再自动切到 packet 视角继续分析。`
- `阅读 Geodo.packet.csv，按企业安全分析的方式给我一份诊断摘要。`

## 第二部分：专业分析 / 字段型问法

这一部分保留给：

- 网络安全分析师
- 懂字段的专业用户
- 开发测试和验收

### 1. flow 级字段问法

- `阅读 BitTorrent.flow.csv，统计 Top 20 的 app_protocol。`
- `阅读 Outlook.flow.csv，统计 Top 10 的 dst_port。`
- `阅读 Gmail.flow.csv，统计 Top 10 的 src_ip。`
- `阅读 Skype.flow.csv，统计 protocol 分布。`
- `阅读 Virut.flow.csv，统计 session_state 分布。`
- `阅读 Zeus.flow.csv，统计 action 分布。`
- `阅读 Outlook.flow.csv，统计 dns_query Top 20。`
- `阅读 Outlook.flow.csv，统计 tls_sni Top 20。`
- `阅读 Virut.flow.csv，统计 http_host Top 20。`

### 2. packet 级字段问法

- `阅读 Virut.packet.csv，统计 TCP flags 分布。`
- `阅读 Zeus.packet.csv，检查 syn-scan。`
- `阅读 Zeus.packet.csv，检查 rst-heavy。`
- `阅读 Virut.packet.csv，检查 handshake-failure。`
- `阅读某个 packet.csv，检查 icmp-probe。`
- `阅读某个 packet.csv，检查 small-packet-burst。`
- `阅读某个 packet.csv，统计 Top 20 的 src_ip。`
- `阅读某个 packet.csv，统计协议分布。`

### 3. SQL 问法

- `阅读 Gmail.flow.csv，执行 SQL：SELECT src_ip, COUNT(*) AS flows, SUM(bytes) AS total_bytes FROM flows GROUP BY src_ip ORDER BY total_bytes DESC LIMIT 10`
- `阅读 Outlook.flow.csv，执行 SQL：SELECT dst_port, app_protocol, COUNT(*) AS flows FROM flows GROUP BY 1,2 ORDER BY flows DESC LIMIT 20`
- `阅读某个 flow.csv，执行 SQL：SELECT dns_query, COUNT(*) AS flows FROM flows WHERE dns_query IS NOT NULL GROUP BY dns_query ORDER BY flows DESC LIMIT 20`
- `阅读某个 packet.csv，执行 SQL：SELECT tcp_flags, COUNT(*) AS packets FROM flows GROUP BY tcp_flags ORDER BY packets DESC`

## 一句话总结

这份清单现在同时保留了：

- **自然用户问法**
- **专业字段问法**

前者适合真实产品使用，后者适合专业分析和测试验收。这样两种使用场景都能覆盖。
